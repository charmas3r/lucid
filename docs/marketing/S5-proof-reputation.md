## 5a · Proof mechanism — every job becomes a public asset

### Case-study template (one per delivered project, lives on `/case-studies`)
The site already renders Sanity case studies with mobile snapshots and a dark theme — this
is a *content ritual*, not a build.

```
TITLE:        [Business], [City] — [outcome in 6 words]
ONE-LINER:    How a [vertical] in [city] went from [before] to [after].
THE NUMBERS:  (capture these — they are the asset)
  - Google rank for "[trade] [city]": #X → #Y
  - Google reviews: N → M  (avg rating A → B)
  - Lighthouse performance: before → 100/100
  - Load time: Xs → Ys  (3–5x faster)
  - Leads/calls or form-fills per month: before → after  (when client shares)
THE STORY:    3 short paras — situation, what Lucid did (Foundation Fix → tier), result.
PROOF SHOTS:  before/after mobile snapshots (existing component), GBP screenshot,
              Search-Console ranking graph.
QUOTE:        one client sentence (collected at the 60-day review ask below).
CTA:          "Want this for your [trade] in [city]? Free audit → 858-215-4894."
```

**Metric-capture rule (non-negotiable):** baseline is recorded *at kickoff* (rank, reviews,
Lighthouse, load time) so every project can become a numbers-backed story later. No
baseline = no provable result. Existing proof assets **Recovery Labz** and **Coin-Plugz**
are the first two studies; the goal is **one new NC case study per closed project**.

### Asset multiplication (one job → many touches)
Each case study spawns: the `/case-studies` page · the proof drop in S4 touch 4 · a GBP
post · a short before/after for Instagram/Nextdoor · a referral-partner talking point (S6).

## 5b · Review system — repeatable ask, timing, target

The ranking lever Evan literally sells (W1) must be run on Lucid itself and every client.

### The review ritual (templated, calendar-driven)
| When | Trigger | Action | Channel |
|------|---------|--------|---------|
| Day 0 | Project kickoff | Set up / verify client GBP review link | — |
| **Day 7 of "wow"** | First visible win (site live / first ranking bump / first lead) | **Ask #1** — "Glad it's working. Would you leave a quick Google review? Here's the direct link." | Text + email |
| Day 14 | No review yet | Friendly reminder w/ the link, 1x only | Text |
| **Day 60** | Results in hand | **Ask #2** — request a one-sentence quote + permission to feature them; re-offer review if still none | Call/email |
| Ongoing | Care plan / retainer | Quarterly: ask the happiest month's client for a review | Email |

**Targets:** every client → **≥1 Google review within 30 days**; **≥1 written testimonial /
case-study quote within 90 days**. For Lucid's own GBP: **+1–2 reviews/month**, ask every
closed client.

**Ask script (copy-paste):** "Hey [Name] — really glad [specific win]. The single biggest
thing that helps a small shop like mine is a quick Google review. Takes 30 seconds: [link].
No pressure, and thank you either way."
